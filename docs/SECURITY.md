# Security Policy (Chính sách Bảo mật)

Thank you for helping us keep our repository and community safe. We take security issues very seriously.
*Cảm ơn bạn đã góp phần giữ an toàn cho kho lưu trữ và cộng đồng của chúng tôi. Chúng tôi luôn coi trọng các vấn đề bảo mật.*

—

## Supported Versions (Các phiên bản được hỗ trợ)

We only provide security updates for the latest stable versions of our tweaks and applications. Please ensure you are running the most recent update before reporting.

*Chúng tôi chỉ cung cấp các bản vá bảo mật cho các phiên bản ổn định mới nhất. Vui lòng đảm bảo bạn đang sử dụng phiên bản mới nhất trước khi báo lỗi.*

| Version / Phiên bản | Supported / Được hỗ trợ | Notes / Ghi chú |
| :— | :—: | :— |
| Latest Release (v2.x) | ❤️ Yes | Active security patches / Được vá lỗi tích cực. |
| v1.x | ❌ No | Deprecated. Please upgrade / Đã cũ, vui lòng nâng cấp. |
| Beta / Nightly | ⚠️ Partial | Only critical vulnerabilities / Chỉ vá lỗi cực kỳ nghiêm trọng. |

—

## Reporting a Vulnerability (Cách báo cáo lỗ hổng bảo mật)

**CRITICAL:** Please DO NOT open a public GitHub Issue for security vulnerabilities or exploits. Doing so risks the safety of all users using this repo.

***QUAN TRỌNG:** Vui lòng KHÔNG mở Issue công khai trên GitHub để báo cáo lỗ hổng bảo mật hoặc lỗi khai thác. Việc này gây nguy hiểm cho toàn bộ người dùng.*

If you discover a security-related bug (e.g., remote code execution, package hijacking, privilege escalation), please report it via one of the following secure channels:

*Nếu bạn phát hiện lỗi liên quan đến bảo mật (ví dụ: thực thi mã từ xa, chiếm quyền gói cài đặt, leo thang đặc quyền), vui lòng báo cáo qua:*

1. **GitHub Private Vulnerability Reporting:** Go to the **Security** tab of this repository -> **Vulnerability reporting** -> Click **Report a vulnerability**. *(Phương thức khuyên dùng)*
2. **Secure Email / Email bảo mật:** [Email_no@example.com]
3. **Encrypted Chat / Chat bảo mật:** Telegram [t.me/kyicchat] (Please state “SECURITY” in your first message).

### What to include in your report / Nội dung cần có trong báo cáo:
* A detailed description of the vulnerability / *Mô tả chi tiết về lỗ hổng.*
* Steps to reproduce or a Proof of Concept (PoC) / *Các bước tái hiện lỗi hoặc đoạn mã chứng minh (PoC).*
* Potential impact (How can an attacker exploit this?) / *Tác động tiềm tàng (Kẻ tấn công có thể lợi dụng lỗi này như thế nào?).*

—

## Our Response Process (Quy trình xử lý của chúng tôi)

Upon receiving your report, we will:
1. Acknowledge receipt of your report within **48 hours**.
2. Investigate the issue and confirm the vulnerability.
3. Work on a fix/patch and coordinate with you to test it.
4. Release the fix and (optional) give credits to you in the `CHANGELOG.md` or Security Advisory, unless you prefer to remain anonymous.

*Sau khi nhận được báo cáo, chúng tôi sẽ:*
1. *Xác nhận đã nhận được báo cáo của bạn trong vòng **48 giờ**.*
2. *Điều tra và xác minh lỗ hổng.*
3. *Tiến hành vá lỗi và phối hợp với bạn để kiểm tra lại.*
4. *Phát hành bản vá và ghi nhận đóng góp của bạn (nếu muốn) trong `CHANGELOG.md`.*

Thank you for your responsible disclosure! / *Cảm ơn bạn đã báo cáo lỗi một cách có trách nhiệm!*
